01 — Overview

A tool used every day by car dealers that made their jobs harder.

During my internship at LuminateOne, I was given the Automote redesign as my core project — a stock and inventory management platform used daily by car dealership staff to track orders, manage vehicle builds, and monitor inventory.

The existing interface had accumulated years of fixes on top of fixes. The result was a tool that was genuinely painful to use. My job was to redesign the core workflows from scratch in Figma, producing a fully interactive prototype that the development team could use to plan implementation.

02 — The Problem

Six problems that made a hard job harder.

Dealership staff use Automote for hours every day. Managing dozens of orders, tracking vehicles through production pipelines, configuring complex builds for customers. Every friction point compounds across hundreds of interactions.

Before designing anything, I spent time understanding the actual workflows — what tasks were done most frequently, where users were getting stuck, and what the existing UI was forcing them to work around rather than with.

  • 01
    Text and buttons cut off

    UI elements truncated at screen edges with no overflow handling. Labels, CTAs, and data values disappeared mid-character.

  • 02
    Navigation required too many steps

    Too many clicks to reach common actions. The hierarchy was flat and unlabelled — users had to memorise locations.

  • 03
    No visual CTA hierarchy

    Primary actions indistinguishable from secondary ones. Everything looked the same weight, unclear what to do next.

  • 04
    Actions completed silently

    No confirmation states, no error messages, no loading indicators. Users couldn't tell if something worked or failed.

  • 05
    Layout broke on different screens

    Built for one fixed resolution. On any other display, components collapsed or overlapped unpredictably.

  • 06
    No consistent design language

    Spacing, type sizes, button styles, and colours varied across screens — making the platform feel unfinished.

03 — Core User Flow

Core workflow: updating order status.

Order status updates were the most frequent dealer action. Previously it required navigating away, making the change, then returning — 5 steps. The redesign brought it to 2.

The redesigned platform.

Order Management

Filterable by ID, customer, model, date, and production status. Clear hierarchy, consistent row actions, and a status system that communicates where each order actually is.

Vehicle Configuration

Two-panel layout keeps build details and configuration steps in view at the same time — removing the need to navigate back and forth mid-task.

06 — Key Design Decisions

The calls I made and why.

🗂

Two-panel layout for configuration

Persistent two-panel view means dealers never lose context when adding or changing build options.

🔍

Collapsible side filter

Filter panel visible when needed, out of the way when not. Filters by ID, customer, model, date, and production status.

🚦

Status as clear text, not icons

Production status displayed as readable text labels, not icons or colour alone. No legend required.

⚓

Structured sidebar navigation

Tools grouped into logical categories (Data Manager, eCommerce, Vehicles) with persistent context.

🎯

Clear primary / secondary action hierarchy

Primary actions use filled buttons with strong colour. Secondary and destructive actions are iconographic and subdued.

📐

Unified component system in Figma

Consistent component library — spacing, type hierarchy, button variants, form elements — applied across every screen.

07 — Testing & Iteration

What the review sessions revealed.

Stakeholder review feedback

The status column wasn't immediately scannable

Dealers scanning a long orders list couldn't quickly identify which orders needed action. All statuses looked visually equivalent.

What I changed

Added visual weight to action-required statuses

Subtle colour coding for statuses requiring dealer action (e.g. "Awaiting Confirmation" in amber, "Ready for Pickup" in green). Text labels retained — colour was additive, not the sole indicator.

Stakeholder review feedback

Bulk actions were missing from the orders view

Sales managers needed to update multiple orders at once — e.g. marking a batch as "In Transit". The v1 design required updating each order individually.

What I changed

Designed a bulk selection and action system

Row-level checkboxes and a contextual bulk action bar that appears when multiple items are selected. Prototyped the full interaction in Figma.

Constraints I worked within

  • Internship timeline — six months meant scoping realistically. Focused on highest-frequency workflows.
  • No direct access to end users (dealership staff) — feedback came through stakeholders, introducing interpretation layer.
  • Prototype for planning only — not involved in implementation, so edge cases needed thorough handoff documentation.
  • Existing branding constraints — visual design had to work within Automote's established colour palette.
08 — Outcomes

How the work was received.

Stakeholders confirmed key pain points were solved

LuminateOne confirmed the redesign directly addressed navigation and clarity issues previously raised by dealership staff. Prototype approved as basis for development planning without further revision cycles.

Reduced cognitive load in high-frequency workflows

The two-panel configure screen and collapsible filter system were specifically called out as solving problems dealers had raised. Status legibility noted as a clear improvement.

Structured prototype reduced development ambiguity

Fully interactive prototype covered all core workflows with annotated specs, interaction states, and documented decisions. Clearer implementation planning, less back-and-forth between design and dev.

09 — Reflection

Designing for people at work.

"B2B design is unforgiving. Consumer apps get forgiven for being confusing — people close them and move on. A tool you use for eight hours a day is different. Every rough edge costs you something."

This project taught me that context of use changes everything. Dealership staff aren't browsing — they're working under time pressure, handling real financial transactions, often across multiple screens. The cost of confusion isn't frustration, it's mistakes with money and customer relationships on the line.

The most valuable lesson was that consistency is a feature. In a complex tool used daily, predictability reduces cognitive load more than any clever interaction. Users should be thinking about their work, not about the interface.

Not having direct access to end users was a real constraint — something I'd push back on in a future project. Stakeholder feedback is useful, but it's a layer removed from the people actually doing the work. Even one session watching a dealer use the existing system would have been worth more than hours of assumption.
